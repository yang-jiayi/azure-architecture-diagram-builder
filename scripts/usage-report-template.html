Data coverage & methodology — read first
  • "Users" = unique anonymous browsers (cookie-based ID), a reach proxy — cleared cookies / incognito / new device each count as a new user, so counts may overstate unique people.
  • Counts use Application Insights dcount() (~1–2% approximation); browser telemetry is not bot-filtered. Volumes are organically distributed, so confidence is high.

Methodology. Data pulled from Azure Application Insights custom events emitted by the Diagram Builder web app, filtered to first-party feature events (Foundry agent traces sharing the resource are excluded). "Users"/"Sessions" via dcount(user_Id) / dcount(session_Id). Verbatim feedback comments are stored in Cosmos DB (not shown here for privacy).
